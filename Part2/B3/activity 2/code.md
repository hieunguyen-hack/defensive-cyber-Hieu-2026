A Content Security Policy (CSP) with a Strict Nonce (Number used once) is the modern gold standard for preventing Cross-Site Scripting (XSS).

Instead of trying to maintain a "whitelist" of safe domains (which is often bypassed by attackers), you generate a unique, cryptographically strong, random string (the nonce) on every single page load.
You tell the browser: "Only execute <script> tags that possess this exact, one-time secret code."

How it works:

Server-side: On every request, the server generates a random string (e.g., ed97asda23).

Header: The server sends a header: Content-Security-Policy: script-src 'nonce-ed97asda23'.

HTML: Only scripts with <script nonce="ed97asda23"> will run. 
If an attacker injects a script via a URL or form field, it won't have the nonce, and the browser will block it.

*CODE:

import secrets
from flask import Flask, render_template_string, make_response

app = Flask(__name__)

@app.route("/")
def index():
    # 1. Generate a unique, cryptographically strong nonce for this specific request
    nonce = secrets.token_urlsafe(16)

    # 2. Mock HTML content
    # Note: The "Attacker's Script" lacks the nonce attribute.
    html_content = f"""
    <!DOCTYPE html>
    <html>
    <head>
        <title>CSP Nonce Demo</title>
    </head>
    <body>
        <h1>Secure Dashboard</h1>
        
        <script nonce="{nonce}">
            console.log("SUCCESS: Legitimate script executed.");
        </script>

        <script>
            alert("XSS ATTACK: I have stolen your cookies!");
            console.error("FAILURE: If you see this, CSP is broken.");
        </script>

        <p>Check the browser console to see the CSP in action.</p>
    </body>
    </html>
    """

    # 3. Create the response and attach the CSP Header
    response = make_response(render_template_string(html_content))
    
    # The 'strict-dynamic' keyword allows nonced scripts to load further dependencies
    csp_header = f"script-src 'nonce-{nonce}' 'strict-dynamic' https:; object-src 'none'; base-uri 'none';"
    response.headers['Content-Security-Policy'] = csp_header
    
    return response

if __name__ == "__main__":
    app.run(debug=True)

The Attack Safeguard:

Even if the attacker updates their malicious link to guess a nonce value, 
like trying to append nonce="abcd" to their payload, 
the browser will see that "abcd" does not match the random token b8X_91aZ_kLp7mQ generated by the backend for that exact session millisecond, and the injection attempt fails.
